### 1. 메인 대시보드 (프로젝트 및 전체 이력 관리 화면)
**필요한 페이지**: 생성된 전체 프로젝트와 업로드된 문서, 생성된 보고서 목록을 통합하여 관리하는 대시보드 페이지입니다.
*   **페이지에 보여야 하는 정보**:
    *   등록된 프로젝트 목록 및 프로젝트 기본 정보 (시나리오 1.1.4).
    *   프로젝트 내 업로드된 원본 문서 목록 및 분석 전처리 상태 (시나리오 3.1.1).
    *   과거에 분석된 테스트 케이스 단건 상세 내역 및 보고서 생성 이력 목록 (시나리오 2.3.3, 3.1.2).
*   **페이지에서 제공해야 하는 Interaction**:
    *   새 프로젝트 생성 버튼을 클릭하여 생성 폼으로 이동 (시나리오 1.1.1).
    *   특정 원본 문서 번호나 분석 단위를 검색 조건으로 입력해 생성 이력을 다중 필터링 (시나리오 3.1.2).
    *   보관 중인 특정 문서 원본 파일 다운로드 (시나리오 3.1.1).
    *   프로젝트 삭제 요청으로 연관 데이터 연쇄 파기 제어 (시나리오 3.2.1).
    *   문서 삭제 시도 시 파생 데이터 파기를 알리는 '삭제 경고 모달 UI' 표출 및 동의 후 삭제 승인 (시나리오 3.2.2).
    *   저장 공간 관리를 위한 과거 검증 보고서 단건 완전 파기/삭제 (시나리오 3.3.1).

### 2. 프로젝트 셋업 화면
**필요한 페이지**: 새로운 분석 대상을 등록하기 위해 프로젝트의 초기 환경을 구성하는 폼 페이지입니다.
*   **페이지에 보여야 하는 정보**:
    *   프로젝트명, 개요, 주요 기능 등을 기재할 수 있는 기본 정보 입력 폼 (시나리오 1.1.1).
    *   분석 대상 리포지토리를 연결하는 GitHub 주소 입력 폼 (시나리오 1.1.2).
*   **페이지에서 제공해야 하는 Interaction**:
    *   프로젝트 기본 정보 폼 기재 및 제출 (시나리오 1.1.1).
    *   GitHub 주소 입력 시 브라우저 단에서 즉각적인 유효성 검사를 수행하여 접근 오류 시 알림 표출 (시나리오 1.1.2).

### 3. 문서 업로드 및 분석 설정 화면
**필요한 페이지**: 분석할 문서를 서버에 전송하고 QA 관점 및 AI 프롬프트 등 분석 파라미터를 설정하는 화면입니다.
*   **페이지에 보여야 하는 정보**:
    *   20MB 용량 제한 및 허용 확장자 안내가 포함된 파일 첨부 영역 (시나리오 1.2.1, 1.2.2).
    *   업로드된 문서 문맥에 맞춰 시스템이 제안하는 '동적 맞춤형 QA 관점 추천' 리스트 (시나리오 1.4.5).
    *   보안, 성능 등 사전 정의된 기본 QA 관점 선택 체크박스 (시나리오 1.4.6).
    *   사용자가 직접 테스트 통제 조건을 지시할 수 있는 사용자 지정 프롬프트 입력창 (시나리오 2.1.2).
    *   파이프라인 시작을 위한 컨텍스트 인식형 원클릭 버튼 ('분석 시작') (시나리오 1.4.9).
*   **페이지에서 제공해야 하는 Interaction**:
    *   파일 선택 시 즉각적으로 용량/확장자를 사전 검사하여 부적합 시 차단 및 알림 (시나리오 1.2.2).
    *   맞춤형 추천이나 기본 QA 관점을 명시적으로 클릭/선택하여 분석 매개변수에 동적으로 반영 (시나리오 1.4.7).
    *   지정 지시(프롬프트) 텍스트 입력 (시나리오 2.1.2).
    *   '분석 시작' 버튼 클릭을 통한 백엔드 비동기 대기열 파이프라인 트리거 (시나리오 1.4.9, 1.4.10).

### 4. 분석 진행 및 결과 시각화 화면 (카드형 대시보드)
**필요한 페이지**: AI 파이프라인 처리 과정을 모니터링하고 생성 완료된 테스트 케이스와 위험 요소, 근거를 확인하는 화면입니다.
*   **페이지에 보여야 하는 정보**:
    *   파이프라인 단계별 진행 상태를 알려주는 모니터링 UI (예: 스피너) (시나리오 1.4.10).
    *   최상단에 독립적으로 배치된 요구사항 명세서 전체 요약 카드 (시나리오 1.7.1.1).
    *   테스트 절차 등이 포함된 구조화된 테스트 케이스 상세 카드 컴포넌트 (시나리오 1.7.1.1).
    *   각 테스트 카드 내부에 부착된 우선순위 배지 및 식별된 위험 요소 해시태그(Tag) (시나리오 1.7.1.2).
    *   자동 축약된(말줄임표) 장문 근거 데이터 텍스트 (시나리오 1.7.1.3).
    *   환각 데이터 제외 등으로 매핑된 근거가 부족한 케이스에 표시되는 '시각적 경고' 마크 (시나리오 2.3.2).
*   **페이지에서 제공해야 하는 Interaction**:
    *   (대기 중) 시스템 통신 상태에 따른 진행 스피너 시각적 피드백 확인 (시나리오 1.4.10).
    *   축약된 장문 근거 하단의 '더 보기' 버튼을 클릭하여 숨겨진 원문을 확장/축소(Toggle)하는 아코디언 상호작용 (시나리오 1.7.1.3).

### 5. 검증 보고서 화면 (미리보기 및 반출)
**필요한 페이지**: 산출된 데이터를 종합한 최종 Markdown 또는 PDF 포맷의 보고서를 열람하고 다운로드하는 화면입니다.
*   **페이지에 보여야 하는 정보**:
    *   한계 고지(Disclaimer), 보완 필요 사항 등이 조립된 최종 보고서 렌더링 화면 (시나리오 1.7.6, 1.7.3.1, 1.7.3.2).
    *   최종 보고서 다운로드 버튼 (시나리오 1.7.7).
*   **페이지에서 제공해야 하는 Interaction**:
    *   별도의 외부 리더기 없이 브라우저 상에서 화면 이탈 없이 최종 보고서 내용 미리보기 검토 (시나리오 1.7.6).
    *   다운로드 버튼 클릭을 통한 보고서 파일 클라이언트 로컬 반출 (시나리오 1.7.7).
    *   파일 유실 상태여도 에러 노출 없이 시스템 내에서 백그라운드 자동 재생성 후 무중단 다운로드 수행 (시나리오 2.3.4).
