## 📘 Phase 2 사용자 테스트 가이드

### 1. 백엔드 및 MinIO 구동
1. 프로젝트 루트 폴더에서 Docker Compose를 사용해 로컬 MinIO를 구동합니다:
   ```bash
   docker-compose up -d
   ```
2. 백엔드 프로젝트(`backend/`)를 구동합니다. H2 DB 파일 모드로 연동되며 포트는 `8080`입니다:
   ```bash
   mvn spring-boot:run
   ```

### 2. 프론트엔드 구동 및 개발 서버 접속
1. 프론트엔드 프로젝트(`frontend/`)에서 의존성을 재설치하고 개발 서버를 시작합니다:
   ```bash
   npm install
   npm run dev
   ```
2. 웹 브라우저를 열어 `http://localhost:5173`에 접속합니다.

### 3. 기능 시나리오 테스트 절차
1. **프로젝트 연동:**
   - 대시보드 상단의 '프로젝트 추가' 버튼을 눌러 프로젝트명과 Public GitHub URL을 입력합니다.
   - 브랜치명(예: `main`)을 입력 후 등록하면, GitHub README.md 수집이 백그라운드에서 실행되며 대시보드로 돌아갑니다.
2. **문서 업로드:**
   - 대시보드에서 프로젝트를 선택하고 '문서 추가' 버튼을 누릅니다.
   - 드래그 앤 드롭 영역에 PDF, TXT, MD, DOCX 파일을 끌어다 놓습니다. 20MB를 초과하거나 HWP 포맷인 경우 브라우저 및 백엔드 단에서 적절히 차단되는지 확인합니다.
3. **분석 시작 & 결과 시각화 (Mock Callback 테스트):**
   - 문서를 업로드한 후, '테스트 케이스 분석하기' 버튼을 누릅니다.
   - 분석 관점 선택 후 '분석 실행'을 클릭하면 비동기 스피너 화면으로 진입하며, 약 8초 후 오프라인 모드(Mock Callback)가 트리거되어 2개의 테스트 케이스를 렌더링하고 우선순위 배지 및 위험 요소 해시태그가 시각화되는지 확인합니다.
4. **보고서 산출 및 다운로드 & 복구:**
   - 우측 상단의 '검증 보고서 산출'을 눌러 Markdown 혹은 PDF를 선택해 생성합니다.
