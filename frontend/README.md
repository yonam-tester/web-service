# 🎨 Yeonam Tester - Frontend Application (React & TypeScript)

연암 테스터(Yeonam Tester)의 사용자 인터페이스(UI)를 담당하는 프론트엔드 모듈입니다. 분석 대시보드, 분석 명세서 관리, 문서 업로드 스텝퍼 및 AI 분석 결과 시각화 카드 등 유려한 글래스모피즘(Glassmorphism) 기반 다크 테마 웹 UI를 제공합니다.

---

## 🛠️ 기술 스택 (Tech Stack)

- **Library**: React 18
- **Language**: TypeScript
- **Bundler & Dev Server**: Vite
- **Styling**: Vanilla CSS & TailwindCSS (글래스모피즘 및 현대적인 다크 모드 스타일 레이아웃 구성)
- **Routing**: React Router DOM (v6)

---

## 🌟 최근 추가 개선 및 확장 스펙 (Phase 5)

개발자 및 사용자가 이해할 수 있는 주요 UI/UX 개선 내역입니다:

1. **분석 진행 중 예외 상황 감지 및 로딩 모달 에러 피드백 UI**
   - AI 분석을 트리거한 후 백엔드를 폴링할 때, 분석 `status === 'FAILED'` 응답이 감지되면 즉시 폴링을 중단합니다.
   - 로딩 모달 하단 영역에 붉은색 에러 경고창 컴포넌트가 조건부 노출되며, 백엔드로부터 전달받은 구체적인 예외 사유(예: "API 키 유효성 검증 실패...")를 렌더링합니다.
   - 에러 박스 내의 "설정 화면으로 돌아가기" 단축 상호작용을 주입해, 사용자가 설정 1단계로 복귀하여 API Key나 설정을 신속하게 변경할 수 있도록 인터랙션을 보강했습니다.
2. **테스트 케이스 상세 카드 영역 2열 다단화 배치**
   - 결과 뷰포트 내의 테스트 케이스 목록 영역을 3열 대시보드 그리드 구조에 간섭을 주지 않는 반응형 2열 구조(`grid grid-cols-1 md:grid-cols-2 gap-6`)로 개편하였습니다.
   - 화면 해상도가 768px 이하(모바일 환경)로 축소될 시, 유연하게 1열 배치 구조로 찌그러짐 없이 반응형 처리되도록 설계되었습니다.
3. **우측 사이드바 실데이터 기획 명세서 누락 목록 렌더링**
   - 우측 패널 '기획 명세서 누락 분석' 영역에 더미 문자열 목록 대신, API 응답 데이터인 `missingItems` 리스트를 바인딩하여 AI 분석을 통해 추출해낸 설계 미흡 텍스트들이 동적으로 한 행씩 아이콘과 함께 표출되도록 완비했습니다.
4. **보안 API Key 입력 및 단일 전송 제어**
   - 비밀번호 입력 마스킹(`<input type="password" />`) 및 드래그 복사 차단 처리가 완비된 API Key 모달을 통해 키를 안전하게 `localStorage`에 영속화합니다.
   - 분석 트리거 시점에만 dynamic API Key를 읽어 백엔드 `llmApiKey` 필드로 실어 전송하는 일원화된 흐름을 처리합니다.

## 🚀 실행 및 빌드 가이드

### 사전 요구 사항
- **Node.js 18** 이상 버전이 필요합니다.
- 백엔드 API 서버(`http://localhost:8080`)가 정상 동작 중인지 확인해 주세요.

### 1. 패키지 의존성 설치
`frontend/` 디렉토리로 이동한 뒤 필요한 패키지를 설치합니다:
```bash
cd frontend
npm install
```

### 2. 로컬 개발 서버 기동
Vite 로컬 개발 서버를 기동하여 변경사항을 실시간으로 확인합니다:
```bash
npm run dev
```
- **접속 주소**: `http://localhost:5173`

### 3. 프로덕션 빌드
프로덕션 배포용 정적 파일을 번들링하기 위해 실행합니다:
```bash
npm run build
```
빌드 완료 시 `dist/` 폴더에 배포용 최적화 파일들이 생성됩니다.

## 🧪 UI/UX 수동 검증 가이드
- **오류 피드백 점검**: 고의로 비정상인 API Key를 등록한 뒤 분석을 구동하여 스텝퍼가 진행되다 멈추는지, 그리고 로딩 모달 내부에 빨간색 에러 메시지 알림 박스와 복귀 버튼이 렌더링되는지 확인합니다.
- **반응형 2열 카드 그리드 확인**: 데스크톱 해상도에서 테스트 케이스가 가로 2열 배치로 깔끔히 정렬되는지 점검하고, 브라우저 창 너비를 최소화했을 때 1열 구조로 유연하게 복원되는지 점검합니다.
- **설계 미흡 항목 실데이터 바인딩 검증**: 분석 완료 후 우측의 기획 명세서 누락 분석 패널에 한 행씩 아이콘 경고 배지와 함께 동적 리스트가 파싱되어 표출되는지 검토합니다.
